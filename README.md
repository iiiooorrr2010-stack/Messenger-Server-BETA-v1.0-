# Messenger-Server-BETA-v2.2

Легковесный мессенджер с асинхронной архитектурой на WebSocket и PostgreSQL.  
Сервер хранит **исключительно не доставленные сообщения**, удаляя их после отправки.  
Проект ориентирован на разработку локальных мессенджеров для малых групп.

## 🚀 Технологии
- **Java 25**
- **Spring Boot 3.4.2** (Web, JPA, WebSocket, Validation)
- **PostgreSQL / H2** - хранение пользователей и оффлайн-сообщений
- **Lombok** - уменьшение шаблонного кода
- **Maven** - сборка
- **Jakarta Validation** - валидация на входе
- **Jetty** - ассинхронность контроллеров на уровне контейнера сервлетов, так же более легковесный, чем tomcat

## 📌 Особенности
- Сообщения хранятся **только до доставки** (оффлайн-буфер)
- ID генерируется по UID
- **Виртуальные потоки** Java 25 для минимальной нагрузки
- Асинхронная обработка WebSocket событий
- Поиск пользователей по имени (отдельный индекс)
- Работа на JWT-токенах(отказались от сессий)

## 🔧 Принцип работы

### 1. Регистрация пользователя
`curl POST ~~~/user/register` - сервер генерирует UID, сохраняет в БД
`curl POST ~~~/user/login` - сервер находит пользователя по индексу и выдает токен регистрации
### 2. WebSocket подключение
Клиент передаёт заголовок сессии, токен пользователя   
Сервер:
- Освобождает сетевой поток мгновенно на уровне Jetty
- В виртуальном потоке обновляет статус пользователя на `ONLINE`
- При обрыве потока по sessionId получаем id и меняем статус на `OFFLINE`

### 3. Отправка сообщения
`@MessageMapping("/chat/send")` → Сервер проверяет статус получателя:
- **Онлайн** → сообщение отправляется мгновенно через STOMP
- **Оффлайн** → сообщение сохраняется в БД до подключения получателя

### 4. Получение оффлайн-сообщений
При подключении пользователь отправляет запрос `@MessageMapping("/chat/get")` →  
Сервер отдаёт все накопленные сообщения и удаляет их из БД.

## 🐞 Обработка ошибок
- `UserNotFoundWithId` (404) — пользователь не найден
- `UsernameExists` (409) — имя уже занято
- `MessageNotFound` (404) — нет непрочитанных сообщений
- `SendToNullUser` (404) — попытка отправить сообщение несуществующему пользователю
- Валидация входных данных через `@Valid`
